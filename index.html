Template Spring MVC

Sources : https://github.com/Swiip/jquery-springmvc-jpa

Il s'agit d'un repository GitHub dans lequel j'essaie de maintenir le plus à jour possible un projet dont la seule règle étant de présenter les données d'une base de données avec un serveur Java et une interface Web.

Le but est de maintenir ce besoin récurent dans mon métier et de faire évoluser les technologies permettant d'y parvenir afin de toujours proposer la meilleur implémentation à un moment donnée.

Genesia

Site communautaire Drupal 7 : http://melserv.org/genesia/

Genesia sur CloudFoundry : http://genesia.cloudfoundry.com/

Sources : seront bientôt publié sur GitHub

Genesia est un projet un peu particulier dans cette liste puisqu'il n'a pas été pensé dans un premier temps pour être Open Source mais plutôt une initiative personnelle dans le cadre de ma passion pour le jeu de rôle.

Il s'agit d'un donc d'un développement de type serveur Web Java dans le but de créer un jeu de rôle où les joueurs incarnent un personnage, passent des ordres sur le sites et échangent entre eux.

Si ce projet apparait dans cette liste, c'est parce qu'il a finit par devenir un projet important pour moi puisqu'il est la mise en pratique de toutes mes préconisations de développement Web Java, sans contraintes professionnelles.

Genesia est lié au projet Template Spring MVC puisqu'il met en oeuvre à l'échelle le template.

Malgré un nombre de lignes de code conséquent, Genesia en tant que jeu n'est pas encore opérationnel

Archetype JavaScript Framework

Sources : https://bitbucket.org/Swiip/archetype-javascript-framework

Archetype est un projet Open Source que j'ai lancé avec Florian Traverse en 2007.

Si le projet est mort aujourd'hui, son objectif est toujours d'actualité : créer un Framework pour le développement d'interface Web en JavaScript qui gère principalement les problématiques d'organisation des chargements des fichiers de l'application Web (JS, CSS, Templates) ainsi que la mise en place d'une structure de composant d'interface respectant les principes du MVC.

Cet objectif était à l'époque très avant gardiste, je peux le dire aujourd'hui sans complexe puisque ces objectifs sont aujourd'hui toujours très d'actualités pour les nouveaux frameworks JavaScript. Nous avons eu des succès et quelques publications et même un contributeur externe malheureusement, le projet n'est jamais allé plus loin. J'analyse aujourd'hui cette perte de vitesse par un manque de communication et de clarté de l'API.

S'il fallait définir aujourd'hui Archetype avec les Frameworks JavaScript existant aujourd'hui, Archetype tendait à apporter la richesse des frameworks RequireJS avec le chargement des dépendances et son compilateur et BackboneJS pour la structure MVC de l'interface Web.

A posteriori, cette experience reste très enrichissante car j'ai la satisfaction de me dire que plusieurs années en avance nos objectifs étaient les bons et l'experience d'avoir développé, communiqué et avoir vécu la gestion d'un projet Open Source.

Lutece

Mise en oeuvre : http://france.meteofrance.com/

Sources : http://dev.lutece.paris.fr/viewvc/lutece/contribs/atoswordline/trunk/

Dans le cadre de mon intervention importante sur la version actuelle du site Web de Météo France, nous avons utilisé le Portail / CMS Open Source Lutece développé par la mairie de Paris.

Suite aux besoins assez particulier en terme de performances du site de Météo France, j'ai été ammené à réaliser des évolutions sur le coeur même de la solution Lutece. Avec l'accord de mon entreprise (Atos WorldLine), j'ai pu reverser les évolutions apportées au projet.

Mon activité professionnelle ne m'a plus permis depuis de travailler dans le domaine des CMS Java mais j'espere que la communauté a pu tirer partie des évolutions.
